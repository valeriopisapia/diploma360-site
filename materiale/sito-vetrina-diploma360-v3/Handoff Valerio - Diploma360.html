1Deploy in 3 passi

È un sito statico puro: nessun build, nessun framework. Va online così com'è.

  1. Pubblica la cartella sito-v3/ su Vercel, Netlify o Cloudflare Pages come sito statico (root = sito-v3).
  2. Imposta le 2 variabili d'ambiente BREVO_API_KEY e BREVO_LIST_ID (per il form — vedi §4) e l'ID GA4 nelle pagine (§5).
  3. Punta il dominio www.diploma360.it e attiva i redirect 301 dal vecchio sito (§6).

In locale: npx serve sito-v3 e apri http://localhost:3000.

Già fatto da me: menu accessibile (tastiera/touch), form con validazione, cookie banner GDPR + Consent Mode v2, lazy-load immagini, sitemap.xml e robots.txt, CSS responsive su tutti i breakpoint, JSON-LD SEO.

2Checklist pre-lancio

Le voci obbligatorio bloccano il lancio; le opzionale si possono fare dopo.

  • API key Brevo + ID lista obbligatorioImposta BREVO_API_KEY e BREVO_LIST_ID come env var. Senza, il form non salva i lead. Vedi §4.
  • ID GA4 reale obbligatorioSostituisci G-XXXXXXXXXX in tutte le pagine con l'ID vero. Vedi §5.
  • Rewrite route pulite obbligatorioCarica vercel.json o _redirects (già scritti). Senza, le 19 landing città danno 404. Vedi §6.
  • Redirect 301 dal vecchio dominio obbligatorioMappa vecchie URL → nuove per non perdere SEO. La founder ti fornisce l'elenco delle vecchie URL.
  • Attributi Brevo PER_CHI, PAGINA_ARRIVO, MESSAGGIO obbligatorioCreali nella lista Brevo prima del lancio, altrimenti vengono ignorati. Vedi §4.
  • Validare pagine legali obbligatorioprivacy.html, cookie.html, termini.html sono BOZZE: falle validare da chi di dovere prima di pubblicare.
  • Verificare invio mail di notifica opzionaleL'endpoint crea il contatto Brevo; se vuoi anche una mail interna a ogni lead, abilita il blocco SMTP commentato in api/lead.js.
  • Submit sitemap in Search Console opzionale/sitemap.xml è pronto: caricalo su Google Search Console dopo il go-live.
  • Foto/loghi reali al posto dei placeholder opzionaleI loghi partner del carosello sono ora quelli ufficiali. Restano da sostituire eventuali foto stock segnalate nel CHANGELOG.

4Form → Brevo

Il form (home + landing) invia un POST JSON a /api/lead. La API key NON sta nel client.

Payload inviato dal client

// js/site.js → fetch POST /api/lead
{
  nome, telefono, email,
  per_chi,      // "Per me" | "Per mio figlio"
  messaggio,    // facoltativo
  pagina,       // path di provenienza (utile per capire da dove arriva il lead)
  ts            // timestamp ISO
}

Cosa fare

  1. In Brevo: crea (o scegli) la lista e annota il suo ID numerico.
  2. Crea gli attributi contatto: NOME, SMS (telefono), PER_CHI, MESSAGGIO, PAGINA_ARRIVO, DATA_RICHIESTA.
  3. Imposta le env var sull'host: BREVO_API_KEY e BREVO_LIST_ID.
  4. Deploy. api/lead.js fa il resto (gestisce anche il contatto duplicato senza errore).
Importante: non mettere mai la API key Brevo nel codice client o in un file committato. Solo come variabile d'ambiente sull'host.
Fallback senza endpoint: se /api/lead non risponde, il form mostra un messaggio con telefono e WhatsApp, così nessun contatto va perso anche prima che tu colleghi Brevo.

5Analytics GA4

Consent Mode v2 è già impostato: GA si carica solo dopo l'accettazione cookie.

In ogni pagina c'è window.__GA_ID__='G-XXXXXXXXXX'. Sostituiscilo con l'ID reale. Find & replace su tutte le pagine:

# dalla cartella sito-v3/
grep -rl "G-XXXXXXXXXX" . | xargs sed -i '' "s/G-XXXXXXXXXX/G-TUOIDREALE/g"

Il default consenso è denied per analytics_storage e ad_storage; passa a granted all'accettazione. Non devi toccare la logica del consenso.

7Come funziona il CSS

  • styles/site.css è l'unica fonte del design system condiviso: token, header/mega-menu, footer, bottoni, card, hero, form, cookie banner, accessibilità, responsive.
  • Ogni pagina linka site.css e ha un piccolo <style> inline con solo il CSS specifico di quella pagina.
  • Token brand canonici: corallo #E48267, gradient #E3815A → #EC89C0, verde AA #16a34a. Modifica i colori solo in site.css.
  • Responsive: breakpoint a 1240 / 1180 / 1000 / 980 / 900 / 860 / 760 / 700 / 640 / 600 / 560 / 520px. Header collassa in menu hamburger sotto 980px; CTA mobile fissa sotto 760px.

8Regole contenuti — non toccare

Decisioni della founder per compliance e tono. Mantienile in qualsiasi modifica.

  • Prezzi 1.500 / 1.900 / 2.900 € — rate 72,68 / 92,06 / 140,52 €/mese × 24. Mai scrivere «senza interessi».
  • Claim onesti: mai «facciamo al posto tuo». Usare «Diploma di Stato riconosciuto» (mai «MIM»). «Coordinatrice del percorso».
  • Niente claim sull'AI — già ripuliti ovunque.
  • Logo PNG ufficiale con «Powered by LaScuola360»; palette cipria.
  • Loghi partner del carosello: ora sono quelli ufficiali forniti dalla founder.
